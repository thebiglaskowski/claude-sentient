# Installation

> Claude Sentient is installed via shell scripts (`install.sh` / `install.ps1`) that copy components, configure hooks with absolute paths, and optionally install LSP plugins. Checksums verify file integrity post-install.

## Installers

| File | Platform | |
|------|----------|-|
| `install.sh` | macOS / Linux / WSL | Bash script |
| `install.ps1` | Windows | PowerShell script |
| `uninstall.sh` | macOS / Linux / WSL | Removes installed files |
| `uninstall.ps1` | Windows | Removes installed files |

**Parity requirement**: `install.sh` and `install.ps1` must list identical components. Integration tests validate parity and fail if the sets diverge.

## Components Installed

| Component | Source | Destination |
|-----------|--------|-------------|
| Commands | `commands/cs-*.md` | `.claude/commands/` |
| Hooks | `.claude/hooks/*.cjs` | `.claude/hooks/` (in-place, already at destination) |
| Utils | `.claude/hooks/utils.cjs` | `.claude/hooks/utils.cjs` |
| Profiles | `profiles/*.yaml` | `profiles/` |
| Rules | `rules/*.md` | `rules/` and `.claude/rules/` |
| Templates | `templates/settings.json` | `.claude/settings.json` (if not exists) |
| Schemas | `schemas/*.json` | `schemas/` |
| Agents | `agents/*.yaml` | `agents/` |
| Native agents | `.claude/agents/*.md` | `.claude/agents/` |

## Absolute Path Patching

Hook commands in `templates/settings.json` use relative paths. At install time, the installer replaces them with **absolute paths** based on the install directory:

**Before (template):**
```json
"command": "node .claude/hooks/session-start.cjs"
```

**After (installed):**
```json
"command": "/path/to/project/node /path/to/project/.claude/hooks/session-start.cjs"
```

This fixes `MODULE_NOT_FOUND` errors when Claude Code is opened from a subdirectory of the project.

### Path Patching Logic

1. At install time: installer reads `settings.json`, replaces `node .claude/hooks/X.cjs` with `{absoluteNodePath} {absoluteHooksDir}/X.cjs`
2. At session start: `session-start.cjs` runs `fixHookPaths()` as a self-healing fallback — re-patches if relative paths are still present (e.g., after a `git pull` that overwrites settings.json)

**Node binary**: Uses `process.execPath` (e.g., `/usr/bin/node` or `/home/user/.nvm/versions/node/v20.x/bin/node`) rather than bare `node`. This bypasses nvm shell function interception that causes FUNCNEST recursion in zsh.

## CHECKSUMS.sha256

`CHECKSUMS.sha256` lists SHA-256 hashes for all installed files. Generated by `generate-checksums.sh`.

### Verification

```bash
sha256sum --check CHECKSUMS.sha256
```

Fails if any installed file has been modified since the checksums were generated.

### Regeneration

Run after any change to installed files:
```bash
bash generate-checksums.sh
```

The DoD verifier hook (`dod-verifier.cjs`) reads this file during integrity checks. It reports mismatches in `.claude/state/last_verification.json` under `integrityChecks`.

## Plugin Auto-Install

The installer auto-installs two categories of Claude Code plugins:

### User-Scoped (Always)
```bash
claude plugin install security-guidance@claude-plugins-official --scope user
```
Installed once at user scope — persists across projects. Not removed by uninstaller.

### Project-Scoped LSP Plugin (Profile-Matched)
The LSP plugin for the detected language profile is installed project-scoped:

```bash
claude plugin install pyright-lsp@claude-plugins-official --scope project
```

| Profile | LSP Plugin |
|---------|-----------|
| python | `pyright-lsp@claude-plugins-official` |
| typescript | `typescript-lsp@claude-plugins-official` |
| go | `gopls-lsp@claude-plugins-official` |
| rust | `rust-analyzer-lsp@claude-plugins-official` |
| java | `jdtls-lsp@claude-plugins-official` |
| cpp | `clangd-lsp@claude-plugins-official` |

Plugin installs are **non-fatal** — failures use `|| true` and don't block setup.

The project-scoped LSP plugin is removed by `uninstall.sh`/`uninstall.ps1`.

## Installation Flags

| Flag | Effect |
|------|--------|
| `--force` | Overwrite existing files; re-patch `settings.json` even if present |
| `--no-plugins` | Skip plugin installation |
| `--dry-run` | Show what would be installed without making changes |

## Business Rules

- **Parity enforcement**: Integration tests compare component lists between `install.sh` and `install.ps1`. Any divergence fails CI.
- **settings.json preservation**: If `.claude/settings.json` already exists, the installer merges hooks config rather than overwriting user customizations.
- **Checksums must be current**: After changing any installed file, regenerate `CHECKSUMS.sha256` before committing. CI verifies checksums are current.
- **Absolute paths required post-install**: Template has relative paths; any settings.json in a real project must have absolute paths. `fixHookPaths()` self-heals on each session start.
- **FORCE_INSTALL flag**: When set, re-applies all patches even if already installed (useful for upgrades).

## Edge Cases

- **Missing `claude` CLI**: Plugin install silently skips. No error.
- **Already installed**: Without `--force`, installer skips files that already exist.
- **Monorepo / subdirectory**: Absolute path patching handles this — hooks resolve correctly regardless of which subdirectory Claude Code was opened in.
- **Windows paths**: `install.ps1` uses `$(Get-Location).Path` and PowerShell path separators. Node binary uses `Get-Command node`.
- **Reinstall after git pull**: If `settings.json` is overwritten by a pull that brings the template, `fixHookPaths()` on next session start re-patches to absolute paths.
