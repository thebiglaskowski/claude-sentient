# Prompt Feedback

This directory stores feedback for continuous prompt improvement.

## Structure

```
feedback/
├── failures/       # When prompts don't work as expected
├── successes/      # When prompts work exceptionally well
└── improvements/   # Drafted improvements based on feedback
```

## File Naming

```
[date]-[prompt-name]-[brief-description].md

Examples:
- 2026-01-29-secure-missed-injection.md
- 2026-01-29-review-excellent-coverage.md
```

## Usage

See `skills/optimization/prompt-feedback.md` for the full feedback workflow.

Quick commands:
- "Record prompt failure" - Create failure report
- "Analyze prompt feedback" - Review patterns
- "Show prompt health" - View statistics
